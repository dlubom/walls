You have not yet compiled the Kakadu documentation

Run "kdu_hyperdoc" using one of the following methods

  • On Unix/Linux/MAC platforms:
    • First invoke the relevant makefile from "coresys/make" to build the core system -- this first step is not strictly necessary, but it allows you to build all applications (not just "kdu_hyperdoc") in the next step.
    • Next, invoke the relevant makefile from "apps/make". This will build all example applications and also build and run "kdu_hyperdoc", generating all HTML documentation and Java language bindings.
  • On Windows platforms with .NET:
    • First open the "coresys/coresys.sln" workspace and navigate to menu item "build->batch build", clicking "rebuild all" after checking both the "debug" and "release" versions -- this first step is not strictly necessary if you only want to run "kdu_hyperdoc", but it is required to build most of the applications later on.
    • Next, open the "apps/apps.sln" workspace and build the applications of interest. To build and run "kdu_hyperdoc", it is sufficient to build only the "kdu_hyperdoc" project in the workspace -- a post-build operation will construct all HTML documentation, plus Java, C# and Visual Basic language bindings. To batch-build all applications, go to menu item "build->batch build" and check only the release versions (fastest and smallest), or only the debug versions, then click "rebuild all".
  • On Windows platforms with Visual C++ version 6:
    • First open the "coresys/coresys.dsw" workspace and navigate to menu item "build->batch build", clicking "rebuild all" after checking both the "debug" and "release" versions -- this first step is not strictly necessary if you only want to run "kdu_hyperdoc", but it is required to build most of the applications later on.
    • Next, open the "apps/apps.dsw" workspace and build the applications of interest. To build and run "kdu_hyperdoc", it is sufficient to build only the "kdu_hyperdoc" project in the workspace -- a post-build operation will construct all HTML documentation, plus Java, C# and Visual Basic language bindings. To batch-build all applications, go to menu item "build->batch build" and check only the release versions (fastest and smallest), or only the debug versions, then click "rebuild all".
  • For more information, read the "Compiling_Instructions.txt" file.
